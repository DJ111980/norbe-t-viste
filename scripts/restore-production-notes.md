# Restore Production Notes

These notes are intentionally manual. Restoring production data must be reviewed before running
commands.

## D1

1. Confirm the target database is `norbe-t-viste-production-db`.
2. Confirm the SQL file is the intended backup.
3. Prefer restoring into a new D1 database first, then validating data.
4. If a direct restore is approved, use Wrangler D1 import/execute according to the current
   Cloudflare documentation and the backup file generated by `scripts/backup-production-d1.ps1`.

## R2

1. Confirm the target bucket is `norbe-t-viste-production-assets`.
2. Restore only keys that belong to production.
3. Use `wrangler r2 object put` for each approved object.

## Safety

- Do not restore staging data into production.
- Do not commit backup files.
- Do not place secrets in the repository.
- Validate login, branding, images, labels, sales, credits and reports after any restore.
